Git adalah sebuah sistem pengontrol versi(Versi Control System) Pada proyek perangkat lunak atau pengembangan perangkat lunak lainnya. Git juga bisa dikenal dengan distributed revision control(VCS terdistribusi)artinya penyimpanan buat database Git tidak hanya berada dalam satu tempat aja. Sehingga semua orang yang terlibat dalam pengkodean proyek akan menyimpan database dalam git, sehingga akan memudahkan dalam pengkodean proyek akan menyimpan database git, sehingga akan memudahkan dalam mengelola sebuah proyek baik secara online maupun offline. Git juga bisa memantau semua perubahan yang terjadi pada file proyek dan menyimpannya ke dalam database. git hanya bisa menyimpan delta perubahannya saja, dia tidak akan menyimpan seluruh isi file yang akan memakan banyak memori.

Kelebihan Git

  • perangkat lunak yang ringan dan cepat
  • git mudah dipahami dalam proyek besar
  • lebih aman digunakan untuk berkolaborasi, karena kita bisa tau apa yang diubah dan siapa yang mengubahnya
  • git bisa menyimpan semua versi file dalam repositori
  • git juga dirancang untuk beroperasi dengan cepat

Command/Perintah pada Git

Git memiliki beberapa alat atau tools yang dapat digunakan untuk melakukan operasi-operasi pada proyek, seperti berikut:

  • Git commit: untuk menyimpan perubahan ke repository lokal
  • Git log: untuk melihat riwayat commit
  • Git branch: untuk melihat seluruh branch yang ada pada resository
  • Git push: untuk mengirimkan perubahan dari repository remote
  • Git add : untuk menambahkan file ke staging area
  • Git clone: untuk membuat salinan repository
  • Git status: untuk melihat status dari file di working directory dan staging area
  • Git init: untuk membuat repository lokal baru
  • Git merge : untuk menggabungkan branch yang aktif dan branch yang dipilih
  • Git checkout : untuk menukar branch yang aktif dengan branch yang dipilih
  • Git diff : untuk melihat perbedaan antara file di working directory dan staging
